优化说明与变更记录

目的

- 允许物体边与墙边严格平行或垂直并且紧贴墙面/物体（允许边界相切）。
- 改善放置精度，减少物品间不必要的间隙，提高利用率。
- 兼顾算法效率，避免网格搜索过粗导致无法贴合。

主要改动

1. 直接使用墙边角度
   - 文件: `TECHNICAL_DOC.md`、`layout.hpp`
   - 说明: 不再对 `wallAngle` 四舍五入到 90° 倍数；`getItemAngleForWall` 返回 `space.wallAngle` 或 `space.wallAngle + 90`，保证物体与墙面严格平行/垂直。

2. SAT 判定允许边界相切
   - 文件: `layout.hpp`（函数 `rectanglesOverlap`）
   - 说明: 将投影分离判断从 `max1 < min2 - EPSILON` 改为 `max1 <= min2 + EPSILON`（并同理处理另一边），使得两个矩形边界正好接触时被视为“非重叠”。

3. 贴墙放置沿墙方向使用向量计算
   - 文件: `layout.hpp`（函数 `tryPlaceAgainstWall`）
   - 说明: 使用 `space.direction` 做沿墙位移，使用 `space.inwardNormal` 做内侧偏移，支持任意斜墙，避免仅对水平/垂直分支处理。
   - 同时，计算哪条边沿墙由 `rot` 决定（`rot==0` -> length 沿墙），而不是比较数值角度。

4. 更细粒度的步长以提高贴合能力
   - 文件: `layout.hpp`
   - 说明: 贴墙步长 `wallStep` 改为 `max(10, min(placeLength, placeWidth)/4)`；网格搜索步长改为 `min(len,wid)/3` 且最小 20。降低步长可以让放置更精确但会增加部分计算量。

5. 日志输出调整
   - 输出放置时使用实际 `itemAngle`（墙角度），便于调试和可视化。

测试结果

- 在 `example1.json` 上进行快速验证：程序成功编译并运行，所有物品放置成功，运行时间 ~31 ms，利用率约 35%。

为什么这些改动有效？

- 直接使用墙角度：当墙是斜的时，roundToNearest90 会将物体旋转到与墙不平行的方向，造成缝隙或重叠。直接采用墙角度保证物体边能严格与墙接触。

- 允许相切：SAT 的经典实现中，间隔边界相等意味着非重叠。原实现用带负容差的不等式把相切当成重叠，导致算法保守，从而留下不必要的缝隙。改为 <=（并带 EPS 容差）可以把相切视为合法放置。

- 向量位移：避免只处理水平/垂直墙的分支，支持任意角度墙，计算更统一、健壮。

- 更细的步长：减少放置位置的离散误差，使得贴墙/贴物更容易实现。但要注意步长太小会放慢搜索。

后续改进建议

- 邻接剪枝：在网格搜索中优先尝试与已有放置物边界对齐的位置（枚举已放置物的边并尝试紧贴），能更快找到紧贴解，减少盲目网格遍历。

- 局部微调：放置成功后，尝试沿两轴做小范围微调（比如 ±step）以寻找更紧密或更高利用率的位置。

- 多线程并行化网格搜索：对大场景可并行尝试不同区域或不同物品旋转方向。

- 可配置步长策略：将步长参数暴露到配置文件或命令行，便于根据场景调优。
